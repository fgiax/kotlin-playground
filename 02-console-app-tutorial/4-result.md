## Main

Gradle по умолчанию задача jar создаёт обычный JAR с классами проекта, а не обязательно исполняемое приложение.

Есть несколько типов JAR:
1. Library JAR — содержит .class-файлы и ресурсы. Его подключают как зависимость к другим проектам. Именно такой JAR ты сейчас получил.
2. Executable JAR — содержит ещё и META-INF/MANIFEST.MF с полем Main-Class, поэтому его можно запускать:

```bash
java -jar app.jar
```

3. Fat (Uber) JAR — помимо твоих классов, включает все зависимости. Это самый удобный вариант для распространения консольных приложений.

## Что получилось в результате `./gradlew run`

Gradle нужен не для "Hello World", а для управления проектом:

- автоматическая сборка;
- подключение библиотек;
- запуск тестов;
- создание JAR;
- публикация артефактов;
- единая сборка для больших проектов.

## Как создать `jar` с помощь `gradle`

Если ты создавал проект через gradle init с плагином application, то есть несколько вариантов.

1. Собрать JAR

./gradlew jar

После этого артефакт появится в:

app/build/libs/

Например:

app/build/libs/app.jar


---

2. Собрать весь проект

./gradlew build

Команда выполнит:

компиляцию;

тесты;

создание JAR.


Артефакт также будет в:

app/build/libs/


---

3. Запустить JAR

Если это обычный JAR с манифестом:

java -jar app/build/libs/app.jar

Но если получишь ошибку вида:

no main manifest attribute

или

ClassNotFoundException

значит был создан обычный JAR, без зависимостей и/или без Main-Class.


---

4. Создать исполняемый ("fat") JAR

Для этого обычно используют плагин Shadow. Тогда достаточно выполнить:

./gradlew shadowJar

и получится файл вроде:

app/build/libs/app-all.jar

который запускается одной командой:

java -jar app/build/libs/app-all.jar


---

Посмотреть, какие задачи доступны

Очень полезная команда:

./gradlew tasks

Она покажет все задачи проекта, например:

Build tasks
-----------
assemble
build
jar

Application tasks
-----------------
run
installDist
distZip
distTar

Если покажешь содержимое твоего app/build.gradle.kts, я скажу, какая команда (jar, build, distZip, installDist или shadowJar) подходит именно для твоего проекта.

## Где артефакты?

После ./gradlew run Gradle не обязан создавать JAR. Он просто компилирует классы и запускает их.

Разберём вывод:

> Task :app:run

Gradle выполнил задачу run для модуля app.

Hello World!

Это вывод твоей программы. Скорее всего, в main() сейчас находится что-то вроде:

fun main() {
    println("Hello World!")
}

Именно эта строка была напечатана в терминал.

BUILD SUCCESSFUL

Сборка и запуск завершились без ошибок.

2 actionable tasks: 2 executed

Gradle выполнил две задачи (например, компиляцию и запуск).

Configuration cache entry stored.

Gradle сохранил кэш конфигурации. При следующем запуске многие этапы будут быстрее.
